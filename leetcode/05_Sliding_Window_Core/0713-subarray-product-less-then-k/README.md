# 713. Subarray Product Less Than K

## 我的思考
这题仍然可以使用滑动窗口，因为 `right` 扩展窗口只会让乘积变大，`left` 收缩窗口只会让乘积变小。

需要思考的是如何收集答案。起初想到也许可以像 992 一样，用“冗余前缀”来统计合法答案。  
但进一步发现，这题更简单一些。

当窗口合法时，**以 `right` 结尾、起点落在当前窗口内的所有子数组都合法**。  
因此问题就变成：窗口每次扩展后，新增了哪些合法子数组？

答案很直接，就是所有**以新元素作为结尾、以窗口内每个位置作为起点**的子数组。  
因此本轮新增答案个数就是：

`right - left + 1`

## 关键点
1. `right` 扩展只会让乘积变大，`left` 收缩只会让乘积变小。
2. 当窗口合法时，以 `right` 结尾、起点在 `[left, right]` 内的所有子数组都合法。
3. 因此每轮直接累加 `right - left + 1`。

## 复杂度
- 时间：`O(n)`
- 空间：`O(1)`

## 注意
1. `k == 0` 或 `k == 1` 时，不存在合法子数组，需要直接返回 `0`。
2. `currProduct` 初始化为 `1`，因为数组元素都大于 `0`。
