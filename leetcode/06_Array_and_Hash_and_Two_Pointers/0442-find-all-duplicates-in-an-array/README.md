# 442. Find All Duplicates in an Array

## 我的思考
初见先想到异或，但很快排除。
之后想到交换排序，这条路是可行的。

初版是直接 `for` 遍历并交换，但很快遇到问题，例如 `[3,1,5,5,1]` 这种情况，会错过 `5`。因为 `5` 被换到了下标 `0`，而遍历已经往后走了，导致它被跳过。

因此后面改成了**链式交换**的思路：  
当当前遍历位置上的值还可以交换，并且交换过去的位置还没有被正确处理、同时当前值也不是答案时，就不能直接遍历下一个位置，而是应该继续处理当前下标。

对应到代码里，就是交换后如果换来的元素原本来自一个还没遍历过的位置，就通过 `i--` 留在当前位置继续处理。

AI 后面给了两个提示，这两个点都值得记忆。

第一，交换排序这类题里，更好的写法通常不是“边交换边收集”，而是**先完成整体位置调整，再统一收集所有不在正确位置上的数**。因为这些不在正确位置上的数，本身就说明它们是重复出现的。这样代码会更简洁、也更易读。

第二，这题还有一个更优的思考角度。  
当 `1 <= nums[i] <= n` 时，可以利用数组下标和符号位做原地标记：

- 访问 `num = abs(nums[i])`
- 去看 `nums[num - 1]`
- 如果该位置已经是负数，说明这个数之前出现过，应当收集
- 如果该位置还是正数，就把它变成负数，表示这个数已经出现过一次

这里的关键在于：  
`abs()` 配合“转负数”这一操作，相当于利用数组元素的正负号额外记录了 `1 bit` 信息，同时又没有丢失原本数值对应的下标含义。

## 卡点
1. 初版直接 `for` 遍历交换，会漏掉交换后落到当前位置的新值。
2. 在交换排序过程中，边交换边收集虽然能做，但流程不够清晰，可读性一般。
3. 一开始没有立刻想到可以利用“正负号”原地做标记。

## 关键点
1. 交换排序类题目里，交换后如果当前位置仍然没有处理完成，就不能急着进入下一个下标。
2. 更清晰的交换排序写法通常是：**先归位，再收集**。
3. 当数组元素范围是 `1 ~ n` 时，可以利用“值映射到下标”的性质做原地标记。
4. `abs(nums[i])` + 把对应位置变为负数，是一种常见且很有价值的 in-place marking 技巧。

## 复杂度
- 时间：`O(n)`
- 空间：`O(1)`，不计答案输出

## 注意
1. 交换排序这类题，容易写出能过但流程较绕的版本；要有意识地区分“调整阶段”和“收集阶段”。
2. `1 <= nums[i] <= n` 这种范围条件非常重要，往往意味着可以把“值”当作“下标”使用。
3. 利用负数空间做标记，本质上是在原数组上增量记录额外信息，这个技巧很值得记忆。
