# 581. Shortest Unsorted Continuous Subarray

## 我的思考
最初的思路是遍历数组并记录当前最大值。  
当当前遍历值小于最大值时，说明这个位置一定落在待排序子数组内。此时还需要继续确定排序起点，也就是在前面已经遍历过的部分中，找到第一个小于等于当前值的位置。

问题在于：如何在 `O(1)` 额外空间下快速找到这个位置。

随后想到，可以持续维护一个“待排序起点”。  
当发现当前值需要排序时：
- 如果当前值仍然应该插入到这个起点之后，那么起点不变，只更新长度
- 如果当前值比起点对应的值还小，就继续向左扩展起点

因此第一版做法是：
- 先找到第一个破坏单调递增的位置
- 设其前一个位置为初始 `begin`
- 后续每遇到一个需要排序的数，就不断向左检查 `begin` 是否还足够靠前
- 同时更新当前待排序区间长度

这版解法可以做到 `O(n) / O(1)`，因为 `i` 最多遍历一遍数组，`begin` 也最多向左移动一遍。

之后又得到第二种思路：  
这题其实不需要精确找出“每个元素应该插入的位置”，本质上只需要找到**待排序区间的左右边界**。

具体来说：
- 从左往右遍历，维护当前最大值  
  如果某个数小于当前最大值，说明它必须被包含在待排序区间中，因此不断更新右边界
- 从右往左遍历，维护当前最小值  
  如果某个数大于当前最小值，说明它也必须被包含在待排序区间中，因此不断更新左边界

最后答案就是：`right - left + 1`

相比第一版，第二版没有再试图模拟元素插入位置，而是直接抓住题目本质，只定位待排序范围，因此结构更清晰、可读性也更好。

## 卡点
1. 初始思路过于关注“当前失序元素应该插到哪里”，导致建模偏细。
2. 虽然第一版复杂度没有问题，但实现上更像过程模拟，可读性一般。
3. 本题真正需要的不是恢复排序过程，而只是定位待排序区间的左右边界。

## 关键点
1. 题目本质是找最短待排序区间，而不是处理每个元素的精确插入位置。
2. 先问题目真正要的是什么结构信息，再决定维护什么。

## 复杂度
- 时间：`O(n)`
- 空间：`O(1)`

## 注意
1. 第一版解法是正确的，但更偏模拟式建模。
2. 第二版更贴合题目本质，也更容易写出简洁、可读性高的代码。
3. 这题暴露出的倾向是：更习惯直接建模和模拟，因此容易先写出能过、复杂度正确的解；但进一步抽象到“题目真正只要求什么”，往往还能继续简化结构。
