# 480. Sliding Window Median

## 我的思考
核心思路：使用两个堆维护中位数  
- 左堆：最大堆  
- 右堆：最小堆  

堆的大小控制为 `k/2` 向上取整，中位数即两个堆顶的平均值  
（当 `k` 为奇数时两个堆顶相同）

滑动窗口本质是两个操作：
- **拓展**：新元素进入窗口
- **删除**：旧元素离开窗口

拓展：
根据新元素与左堆顶比较，决定加入左堆还是右堆。

删除：
堆的随机删除复杂度为 `O(k)`，因为 `PriorityQueue` 删除指定元素需要先搜索。

因此需要避免真正删除元素。

解决方法：**延迟删除（lazy deletion）**

- 使用 `Map` 记录待删除元素及其数量
- 当元素出现在堆顶时再真正删除

同时需要手动维护两个堆的**真实高度**。

### 插入策略
比较新元素与左堆顶：
- `<= 左堆顶` → 左堆
- `> 左堆顶` → 右堆

并手动更新高度。

### 删除策略
- 将删除元素记录到 `delayCount`
- 判断该元素属于哪个堆
- 手动减少对应堆高度

### 中位数计算流程
1. **平衡堆高度**  
   两堆高度差不能超过 1  
   （平衡时需要先清理堆顶的延迟删除元素，否则高度会错误）

2. **prune 清理堆顶**  
   如果堆顶元素在 `delayCount` 中，则删除

3. **计算中位数**  
   - 两堆等高 → 平均值  
   - 否则 → 高堆堆顶

## 关键点
- `PriorityQueue` 不支持 `O(log n)` 删除指定元素
- 使用 **延迟删除（lazy deletion）**
- 需要手动维护堆的真实高度

## 复杂度
- 时间：`O(n log k)`
- 空间：`O(k)`

## 注意
可以规定左堆始终是较高的堆（或与右堆等高），  
这样可以简化部分判断逻辑，但在 `balance` 阶段仍然需要双向调整，因此没有强制使用这一约束。
