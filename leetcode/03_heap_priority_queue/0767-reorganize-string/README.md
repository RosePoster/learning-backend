# 0767. Reorganize String

## 我的思考
可视为“冷却期为 1 的调度问题”：
每次优先选择剩余次数最多的字符，
并延迟上一轮使用的字符 1 轮后再入堆。

→ 优化1：字符范围固定 26，使用 int[26] 代替 Map
→ 优化2：使用 DummyIndex 作为初始 preC，避免额外判断

## 关键点
- 贪心：每次取当前频率最大字符
- 延迟机制：上一轮字符暂存一轮，防止相邻重复
- 若堆为空但仍未完成 → 无法构造

## 复杂度
- 时间：O(n log 26)
- 空间：O(26)

## 注意
本题仍然有一个数学解，统计频率最大的字母，填到偶数位上，剩余的字母依次继续填偶数位，偶数位填完再填奇数位即可
