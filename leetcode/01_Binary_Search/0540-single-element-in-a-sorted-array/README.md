# 540. Single Element in a Sorted Array

## 我的思考
这题的思路是：把相等数字看作一个 `pair`，而唯一答案相当于独占一个位置。

解法维护这样一个不变量：

- 区间 `[begin, mid]` 内是完整的 `pairs`
- 区间 `(mid, end]` 内是完整的 `pairs`

然后通过判断 `begin` 到 `mid` 这一段长度的奇偶性，来判断答案落在哪一边。

实现中的关键处理是 `mid--`。  
在这一步之后，`mid` 不可能等于 `begin - 1`，因此没有越界风险。理由是：

- 数组总长度为奇数；
- 当数组长度为 `3` 及以上时，这里不会出现越界；
- 当数组长度为 `1` 时，循环根本不会进入，因为只有 `begin < end` 才会进入循环。

因此，这个写法是安全的。

AI 的评价是：主流答案通常通过判断 `mid` 下标的奇偶性来做，这个解法与主流解法本质相同，只是切入角度不同。

## 关键点
1. 本质上仍然是利用“答案出现后，配对结构整体错位”这一性质，只是没有直接从下标奇偶性切入，而是从“完整 `pair` 区间”的角度切入。

## 复杂度
- 时间：`O(log n)`
- 空间：`O(1)`

## 注意
1. `if(nums[mid] != nums[mid - 1]) mid--;` 这一步的作用，是把 `mid` 统一调整到当前 `pair` 的右端点，便于后续通过左侧长度奇偶性判断答案位置。
2. 这里的核心不是单纯比较下标奇偶，而是维护“某一侧是否由完整 `pairs` 构成”的结构性不变量。
3. 最后循环收敛到 `begin == end` 时，`nums[begin]` 就是答案。
