# 421. Maximum XOR of Two Numbers in an Array

## 我的思考
本题花费 2h，标记为重做。

首先是思路过程（1h）：

第一眼没有什么思路。  
题目意思是：数组中任意两个不相等的数，返回它们异或后的最大结果。

暴力解是 `O(n^2)`，不考虑。

仅遍历一遍的 `O(n)` 能做到吗？也许可以？  
但大概率是 `O(n log n)`，于是先想 `O(n log n)` 的解法。

异或本质上是不进位加法。  
要找到最大的结果，需要让最高位的 `1` 不能被消去。

但这能保证最大吗？不能。  
假设多个数都有这个最高位的 `1`，就无法仅凭这一点确认最大值。

如果最高位相同的数有多个，那么就得继续判断次高位。  
不但要让原有位不能消去，还要考虑通过异或引入次高位 `1` 的情况。

这样看的话，也许可以把数分组。  
假设某个数是 `101101`，那么这个数就属于 `1、3、4、6` 号数组。  
产生答案的两个异或数字中，其中一个一定出自序号最大的数组。

但其实没必要先完整分组再找。  
实际上可以用贪心解法，因为异或不进位，高位有 `1` 就一定比低位大。

于是问题转化为：

**假设已经确认了高 `k` 位，现在要确定第 `k+1` 位能不能为 `1`，要怎么判断？**

AI 提示：给定数组里所有数的高 `k` 位前缀，如何快速判断“存在两个数，它们的前 `k+1` 位异或结果的最高位是 `1`”？

一开始理解错了。  
当时想到：只需要将得出这个高 `k` 位答案候选者根据第 `k+1` 位的值分为 `0` 组和 `1` 组。  
一旦某一组没有数了，这一位就只能是 `0`。  
否则这一位就可以是 `1`，答案是两组间各取一个数的异或，然后继续往后判断。

但再下一位就会变复杂：  
上一次分组时分在同组的两个数不能同时取；  
上一次分在不同组、这一次又分在同组的两个数也不能同时取。  
也就是说，必须始终分在不同组的数才能作为答案。

这样开销很大，逻辑也很难实现。

AI 提示：不是这个理解。

接着想到：要用 Trie。  
将数组根据比特位转化为 Trie 树。

然后对于每一个数，都去搜索一遍 Trie。  
搜索时优先走当前比特位相反的方向，这样就能找到这个数可以得到的最大异或值。  
再用 `ans` 收集所有数各自最大异或值中的最大值。

时间复杂度是 `O(n * k)`，其中 `k` 为 Trie 的深度，也就是数组最大值的二进制位数。

并且从高位往低位找时，一旦某一个数当前这一位只能取到 `0`，而即使把后面所有位都补成 `1` 也不可能超过当前全局最大答案，就可以提前退出这个数的搜索。

到这里，思路就已经收敛了，而且是最优解。

然后开始实现（耗时 1h）：

出现了两个问题：

1. `<<` 和 `>>` 这两个操作还不够熟练。
2. 对 Trie 节点和边的关系把握还不够稳定，总是担心当前节点存的是值，还是边存的是值。

并且一开始没有区分 `Trie` 和 `TrieNode`，把这二者和 `maxBitPos` 混在一起了。

以下这些操作，都是知道要这样做，但一开始不知道怎么高效做，属于 AI 提示的内容：

1. `(currMaxXor | ((1 << i) - 1))`  
   知道这里可以剪枝，但不知道怎么高效判断。
2. `int nBit = bit ^ 1`  
   知道这里要取反，但不知道怎么高效取反。
3. `this.maxBitPos = (maxNum == 0) ? 0 : 31 - Integer.numberOfLeadingZeros(maxNum);`  
   知道这里要找最大比特位，但不知道怎么快速找到。
4. `<<` 和 `>>` 的操作  
   一开始完全没有想到可以这样处理比特。

总结：

思路部分，一旦想到 Trie，就可以非常顺畅地收敛到最优解。  
实现部分，说明比特操作还需要继续锻炼。

## 卡点
1. 一开始没有找到有效的建模方式，在“按位分组”这条路上绕了较久。
2. 对“逐位贪心判断答案是否可行”有感觉，但没有找到一个低开销、可实现的数据结构来承载这个过程。
3. 实现时对位运算不够熟练，尤其是移位、取反、求最高有效位这些细节不够顺手。
4. `Trie`、`TrieNode`、树节点含义、边的含义这几个概念一开始有些混淆。

## 关键点
1. 异或比较大小时，本质上是高位优先，高位一旦更大，低位就不重要。
2. 用二进制 Trie 表示所有数，可以把“找某个数的最佳异或对象”转化为“每一位尽量走相反分支”。
3. 对于当前数的某一位 `bit`，优先走 `bit ^ 1`，因为这样该位异或结果为 `1`，更优。
4. Trie 的深度只需要到数组最大值的最高有效位，不必固定走满 31 位。
5. 可以做剪枝：若当前已得到的 `currMaxXor` 即使后续全部补 `1` 也无法超过 `ans`，则可以提前停止当前搜索。

## 复杂度
- 时间：`O(n * k)`，其中 `k` 是数组最大值的二进制位数
- 空间：`O(n * k)`

## 注意
1. `TrieNode` 表示节点本身，`next[0]` 和 `next[1]` 表示沿着二进制位 `0/1` 的下一条边，不要把“节点存值”和“边表示位”混在一起。
2. `int bit = (num >> i) & 1` 表示取出 `num` 的第 `i` 位。
3. `int nBit = bit ^ 1` 可以在 `0/1` 之间快速切换，表示当前位的相反位。
4. `1 << i` 表示第 `i` 位为 `1` 的数值。
5. `(currMaxXor | ((1 << i) - 1))` 表示“当前结果不变，但把后面低位全部看作 `1` 时所能达到的理论最大值”，可用于剪枝判断。
6. `31 - Integer.numberOfLeadingZeros(maxNum)` 用于求一个非负整数的最高有效位位置。
7. 当 `maxNum == 0` 时要单独处理，否则最高位计算会出问题。
